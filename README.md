                                                                    Архитектура и технологии
                                                Приложение построено по клиентской (client-side) архитектуре.
    Используемый стек:
        Frontend: HTML, CSS, JavaScript.
        Backend: Отсутствует. ( Вся логика выполняется в браузере пользователя, поэтому Backend не требуется )
    Анализ уязвимостей
        В текущей реализации приложение полностью безопасно, поскольку оно является статичным: не принимает, не хранит и не обрабатывает никаких пользовательских данных.
    Однако при дальнейшем усложнении проекта могут возникнуть гипотетические риски:
        Межсайтовый скриптинг (XSS): Уязвимость может появиться, если добавить функционал для ввода и отображения данных от пользователя без их должной обработки (санитизации).
        Уязвимости сторонних библиотек: Риск возникнет при подключении внешних библиотек (фреймворков), если их не обновлять до актуальных версий.
Вывод: Приложение безопасно благодаря своей простоте и отсутствию обработки внешних данных.
